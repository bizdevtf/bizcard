# 아이폰 단축어: 명함 사진 → 연락처 저장

카메라로 명함을 찍고 **공유하기 → "명함 저장" 아이콘 탭** 한 번으로,
Claude가 명함을 인식해 연락처 저장 화면까지 띄워주는 iOS 단축어 설정 가이드.

동작 원리: 단축어가 사진을 Claude API로 직접 보내고, Claude가 vCard(.vcf) 텍스트로
응답하면 단축어가 이를 파일로 만들어 연다. (Claude 앱을 열 필요 없음)

## 사전 준비: API 키 발급 (1회)

1. https://console.anthropic.com 접속 → 가입/로그인
2. 결제 수단 등록 (사용한 만큼만 과금 — 명함 1장당 약 10~20원 수준)
3. **API Keys** 메뉴 → **Create Key** → 키 복사 (`sk-ant-...` 형태)
   - 이 키는 비밀번호와 같으니 다른 사람과 공유하지 말 것

## 단축어 만들기

아이폰 **단축어(Shortcuts)** 앱에서:

### 1. 새 단축어 + 공유 시트 설정

1. 단축어 앱 → 우상단 `+` → 새 단축어
2. 상단 이름을 **"명함 저장"**으로 변경, 아이콘/색상 선택 (예: 명함 모양)
3. 상단 정보(ⓘ) → **"공유 시트에 표시"** 켜기
4. 받는 콘텐츠 유형: **이미지**만 선택

### 2. 동작(액션) 추가 — 아래 순서대로

**① 이미지 변환**
- 동작 검색: "이미지 변환"
- `단축어 입력`을 **JPEG**로 변환 (품질: 중간)
- ※ 아이폰 기본 HEIC 형식은 API가 못 읽으므로 필수. 용량도 줄어듦.

**② Base64 인코딩**
- 동작 검색: "Base64 인코딩"
- 대상: `변환된 이미지`
- 옵션 펼쳐서 **줄바꿈: 없음** 으로 설정 (중요 — 줄바꿈 있으면 API 오류)

**③ 텍스트** (API 요청 본문)
- 동작 검색: "텍스트", 아래 내용 입력.
- `[BASE64]` 자리에는 ②의 결과 변수(`Base64 인코딩됨`)를 삽입한다.

```json
{
  "model": "claude-opus-4-8",
  "max_tokens": 1024,
  "messages": [
    {
      "role": "user",
      "content": [
        {
          "type": "image",
          "source": {
            "type": "base64",
            "media_type": "image/jpeg",
            "data": "[BASE64]"
          }
        },
        {
          "type": "text",
          "text": "이 명함 이미지에서 연락처 정보를 추출해 vCard 3.0 형식으로만 응답하라. BEGIN:VCARD로 시작해 END:VCARD로 끝나야 하며, 다른 설명이나 코드블록 표시는 절대 넣지 마라. 규칙: 한글/영문 병기 시 한글 이름을 FN에 쓰고 영문 이름은 NOTE에 넣는다. 전화번호는 하이픈 포함으로 정규화한다(예: 010-1234-5678). 항목: N, FN, ORG(회사;부서), TITLE, TEL;TYPE=CELL, TEL;TYPE=WORK, EMAIL, ADR, NOTE. 읽을 수 없는 항목은 생략한다."
        }
      ]
    }
  ]
}
```

**④ URL 콘텐츠 가져오기**
- URL: `https://api.anthropic.com/v1/messages`
- 방법: **POST**
- 헤더 3개 추가:
  | 키 | 값 |
  |---|---|
  | `x-api-key` | (발급받은 API 키) |
  | `anthropic-version` | `2023-06-01` |
  | `content-type` | `application/json` |
- 본문 요청: **파일** → ③의 `텍스트` 변수 선택

**⑤ 사전 값 가져오기** — `content` 가져오기 (대상: `URL 콘텐츠`)

**⑥ 목록에서 항목 가져오기** — **첫 번째 항목**

**⑦ 사전 값 가져오기** — `text` 가져오기 (대상: ⑥의 항목)
- 이 값이 vCard 텍스트다.

**⑧ 이름 설정** — ⑦의 결과 이름을 `연락처.vcf`로 설정
- "확장자 포함" 관련 옵션이 보이면 켠다.

**⑨ 빠른 보기(Quick Look)**
- 미리보기가 뜨면 연락처 카드가 보이고, 공유 버튼 → **연락처에 저장** 가능.
- ※ 기기/iOS 버전에 따라 ⑨ 대신 **"공유"** 동작을 넣고 공유 시트에서
  "연락처" 앱을 선택하는 방식이 더 매끄러울 수 있다. 둘 다 시험해볼 것.

### 3. 사용법

1. 카메라(또는 사진 앱)에서 명함 사진 → **공유** 버튼
2. 공유 시트에서 **"명함 저장"** 탭
3. 몇 초 후 연락처 카드가 뜨면 **저장**

## 비용/모델

- 기본 모델은 `claude-opus-4-8` (인식 정확도 최우선). 명함 1장당 약 10~20원.
- 비용을 낮추려면 ③의 `"model"` 값을 `"claude-haiku-4-5"`로 바꾼다.
  1장당 약 2~4원 수준이며 일반적인 명함 인식에는 충분한 경우가 많다.

## 문제 해결

| 증상 | 원인/해결 |
|---|---|
| `invalid_request_error` (400) | Base64에 줄바꿈 포함 → ②에서 "줄바꿈: 없음" 확인 |
| `authentication_error` (401) | API 키 오타 또는 헤더 키 이름(`x-api-key`) 확인 |
| 응답이 비었거나 이상함 | ⑤~⑦ 경로 확인: `content` → 첫 항목 → `text` |
| 연락처 카드가 안 뜸 | ⑧ 파일 이름이 `.vcf`로 끝나는지 확인, ⑨를 "공유"로 교체해 시험 |
| HEIC 관련 오류 | ① 이미지 변환(JPEG) 단계 누락 |
